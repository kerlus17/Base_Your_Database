## Task

### 1. Read the Problem
- Open the `problem.pdf` file.
- Carefully read and understand the requirements before starting.

### 2. Draw the Entity-Relationship Diagram (ERD)
- Create the ERD based on the problem description.
- Clearly show entities, attributes, primary keys, relationships, and cardinalities.
- You may use the draw.io extension in VS Code or the website or any other website you like.

### 3. Create the Schema
- Convert the ERD into a relational schema.
- Specify table names, attributes, primary keys (PK), and foreign keys (FK).
- This version should represent the schema **before normalization**.

### 4. Normalize the Schema
- Normalize the schema (up to 3NF unless stated otherwise).
- Produce the final schema **after normalization**.

### 5. Submission
- Create a folder named `BYD_session_2`.
- Include three draw.io files or images:
  - ERD
  - Schema before normalization
  - Schema after normalization
- Submit the folder as instructed.
